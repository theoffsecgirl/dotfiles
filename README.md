## What is this

These are not "pretty dotfiles".

This is my real bug bounty working environment.

Designed to:

- stop wasting time on setup
- keep the same workflow everywhere
- automate repetitive tasks
- go straight to finding bugs

## Setup

```bash
git clone git@github.com:theoffsecgirl/dotfiles.git ~/.dotfiles
cd ~/.dotfiles
./install.sh
exec zsh
```

If something breaks:

```bash
hunt-doctor
```

## Bug bounty workflow

### 1. Create target

```bash
mktarget example.com
```

Official target layout:

```text
$HUNTING_HOME/targets/example.com/
├── recon/       # subdomains, scope files and recon output
├── http/        # live hosts, crawled URLs and HTTP metadata
├── fuzz/        # parameters and fuzzing candidates
├── js/          # JavaScript files
├── in/          # input files such as resolvers
│   └── resolvers.txt
├── out/         # generic command output
├── tmp/         # temporary files
├── burp/        # Burp exports or notes
├── notes/       # target notes
│   └── summary.md
├── reports/     # reports or drafts
├── loot/        # relevant evidence
└── meta/        # JSON metadata generated by tools
```

### 2. Enumerate subdomains and live HTTP services

```bash
scope example.com
```

Main outputs:

```text
recon/subdomains.txt
http/live.txt
http/httpx.jsonl
http/httpx_table.tsv
meta/scope.json
```

### 3. Crawl and map URLs

```bash
webmap example.com
```

Main outputs:

```text
http/urls.txt
http/urls_clean.txt
http/api_candidates.txt
http/graphql.txt
js/files.txt
meta/webmap.json
```

### 4. Extract parameters

```bash
paramhunt-v2 example.com
```

Main outputs:

```text
fuzz/urls_with_params.txt
fuzz/params.txt
meta/paramhunt.json
```

### Recommended flow

```bash
mktarget example.com
scope example.com
webmap example.com
paramhunt-v2 example.com
```

Expected command resolution:

```bash
type -a scope webmap mktarget subscan
```

All four should resolve to `~/.local/bin/*`.

## Quick navigation

[`zoxide`](https://github.com/ajeetdsouza/zoxide) learns your most used directories:

```bash
z hunting       # jump to hunting workspace
z dotfiles      # jump to ~/.dotfiles
zi              # interactive selector with fzf
dotfiles        # direct alias to ~/.dotfiles
hunting         # direct alias to $HUNTING_HOME
```

Bug bounty helpers:

```bash
cdh        # cd $HUNTING_HOME
cdt        # cd $HUNTING_HOME/targets
cdn        # cd $HUNTING_HOME/notes
cds        # cd $HUNTING_HOME/scripts
```

## Tools

Tool installers live in:

```text
tools/install-tools.sh
tools/update-tools.sh
```

Generated binaries and cloned sources are intentionally ignored:

```text
tools/bin/
tools/src/
```

Rebuild local tools with:

```bash
bash ~/.dotfiles/tools/install-tools.sh
```

Update them with:

```bash
bash ~/.dotfiles/tools/update-tools.sh
```
